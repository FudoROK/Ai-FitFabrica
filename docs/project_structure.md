## Часть 2: Подробное описание структуры проекта (папка `src`)

### `src/`

Корневая директория исходного кода приложения. Содержит основные модули и компоненты, из которых строится бэкенд AI-ассистента.

*   **`__init__.py`**:
    *   **Назначение**: Маркирует директорию `src` как пакет Python. В данном случае файл пуст, что означает стандартную инициализацию пакета без выполнения дополнительного кода при импорте.
    *   **Что в нем**: Обычно пуст или содержит базовые импорты для пакета.
*   **`main.py`**:
    *   **Назначение**: Основной исполняемый файл приложения. Это точка входа для FastAPI-приложения, где определяется экземпляр FastAPI и, вероятно, подключаются корневые маршруты или другие компоненты инициализации.
    *   **Что в нем**: Код, запускающий сервер, и, возможно, базовая конфигурация FastAPI.
*   **`settings.py`**:
    *   **Назначение**: Централизованное хранилище всех конфигурационных настроек приложения. Здесь определяются переменные окружения, ключи API, параметры подключения к базам данных и другие изменяемые параметры, необходимые для работы приложения в различных средах (разработка, тестирование, продакшн).
    *   **Что в нем**: Классы или объекты, загружающие настройки из переменных окружения (например, с использованием `pydantic.BaseSettings`).

### `.adk/`

*   **Назначение**: Эта директория, вероятно, предназначена для хранения артефактов, связанных с ADK (Agent Development Kit). Она может содержать сгенерированные файлы, временные данные или кэш, используемые в процессе разработки или выполнения агентов.
*   **Что в нем**: Чаще всего это специфичные для инструмента или фреймворка файлы, которые не являются частью основной логики приложения, но поддерживают его работу.

### `adapters/`

Этот слой отвечает за взаимодействие с внешним миром. Он содержит реализации "портов" из доменного слоя, переводящие доменные концепции в специфические вызовы сторонних API или баз данных. Если нужно изменить CRM, базу данных или провайдера LLM, изменения будут производиться здесь.

*   **`__init__.py`**: Маркирует `adapters` как пакет и, возможно, экспортирует некоторые общие адаптеры.
*   **`calendar/`**:
    *   **Назначение**: Содержит адаптеры для интеграции с внешними календарными сервисами. Позволяет приложению создавать, читать, обновлять или удалять события в календарях пользователей.
    *   **`__init__.py`**: Инициализация пакета.
    *   **`calendar_adapters.py`**:
        *   **Назначение**: Реализация конкретных адаптеров для работы с календарными API (например, Google Calendar, Outlook Calendar). Здесь будет код для аутентификации, форматирования запросов и обработки ответов от календарных сервисов.
        *   **Что в нем**: Классы и функции, преобразующие доменные запросы к календарю в специфичные для API вызовы.
*   **`crm/`**:
    *   **Назначение**: Адаптеры для взаимодействия с различными системами управления взаимоотношениями с клиентами (CRM). Позволяет получать и обновлять информацию о лидах и клиентах.
    *   **`__init__.py`**: Инициализация пакета.
    *   **`hubspot/`**:
        *   **Назначение**: Специфические адаптеры для интеграции с CRM-системой HubSpot.
        *   **Что в нем**: Классы и функции для взаимодействия с API HubSpot.
    *   **`no_crm/`**:
        *   **Назначение**: Заглушка или "нулевой" адаптер CRM, который используется, когда интеграция с реальной CRM не требуется или отключена. Это позволяет системе работать без ошибок, даже если CRM-функциональность не активна.
        *   **Что в нем**: Простые реализации интерфейсов CRM, которые ничего не делают или возвращают пустые/дефолтные значения.
*   **`database/`**:
    *   **Назначение**: Адаптеры для работы с различными базами данных. Абстрагирует доменный слой от деталей конкретной СУБД.
    *   **`__init__.py`**: Инициализация пакета.
    *   **`firestore/`**:
        *   **Назначение**: Адаптеры для Google Cloud Firestore, NoSQL базы данных. Позволяет сохранять и извлекать данные.
        *   **Что в нем**: Классы-репозитории, использующие клиентскую библиотеку Firestore для выполнения операций CRUD (создание, чтение, обновление, удаление).
*   **`external_apis/`**:
    *   **Назначение**: Адаптеры для общих внешних API, которые не подпадают под более специфичные категории (CRM, LLM, Messaging).
    *   **`__init__.py`**: Инициализация пакета.
    *   **`google_maps/`**:
        *   **Назначение**: Адаптеры для взаимодействия с API Google Maps (например, для геокодирования, определения часовых поясов).
        *   **Что в нем**: Классы, выполняющие HTTP-запросы к API Google Maps и парсящие их ответы.
*   **`factories/`**:
    *   **Назначение**: Содержит фабрики, которые отвечают за создание и конфигурирование экземпляров адаптеров и других зависимостей приложения. Это помогает управлять инверсией управления и внедрением зависимостей.
    *   **`__init__.py`**: Инициализация пакета, может экспортировать фабрики.
    *   **`llm_provider_factory.py`**:
        *   **Назначение**: Фабрика для создания экземпляров различных провайдеров LLM (например, Gemini, OpenAI) на основе конфигурации.
        *   **Что в нем**: Функция или класс, который принимает параметры и возвращает соответствующий объект провайдера LLM.
    *   **`runtime_dependency_factory.py`**:
        *   **Назначение**: Фабрика для создания зависимостей, необходимых во время выполнения приложения (например, клиентов баз данных, сервисов, лимитеров скорости).
        *   **Что в нем**: Методы, инстанцирующие и конфигурирующие объекты, которые затем внедряются в другие части приложения.
*   **`llm/`**:
    *   **Назначение**: Адаптеры, специфичные для взаимодействия с провайдерами больших языковых моделей (LLM). Отдельно от `llm` в корне `src`, этот `llm` в `adapters` может содержать более низкоуровневые или специфичные для провайдера обертки.
    *   **`__init__.py`**: Инициализация пакета.
*   **`messaging/`**:
    *   **Назначение**: Адаптеры для интеграции с различными платформами обмена сообщениями.
    *   **`__init__.py`**: Инициализация пакета.
    *   **`telegram/`**:
        *   **Назначение**: Специфические адаптеры для взаимодействия с API Telegram (отправка/получение сообщений, обработка вебхуков).
        *   **Что в нем**: Классы, реализующие логику работы с Telegram Bot API.
    *   **`messaging_adapters.py`**:
        *   **Назначение**: Общая реализация адаптеров для обмена сообщениями, которая может быть использована различными мессенджерами, или же это основной файл для общих функций обмена сообщениями.
        *   **Что в нем**: Классы и функции для отправки и получения сообщений через унифицированный интерфейс.

### `adk_agents/`

Этот пакет содержит специфические реализации агентов, использующих Agent Development Kit. Эти агенты инкапсулируют сложную логику, часто взаимодействуя с LLM и управляя различными аспектами поведения системы.

*   **`__init__.py`**: Маркирует `adk_agents` как пакет и, возможно, экспортирует базовые классы агентов.
*   **`daily_memory_agent/`**:
    *   **Назначение**: Агент, который управляет "ежедневной" памятью ассистента. Это может включать суммирование взаимодействий за день, сохранение ключевых фактов или выполнение задач, привязанных к концу дня.
    *   **`__init__.py`**: Инициализация пакета.
    *   **`agent.py`**:
        *   **Назначение**: Основная логика агента ежедневной памяти. Определяет, как агент обрабатывает входные данные, взаимодействует с LLM и сохраняет результаты, связанные с ежедневным циклом.
        *   **Что в нем**: Класс агента, методы обработки, вызовы LLM и логика сохранения.
    *   **`contracts.py`**:
        *   **Назначение**: Определяет интерфейсы и модели данных (контракты), с которыми работает агент ежедневной памяти, как для входящих данных, так и для исходящих результатов.
        *   **Что в нем**: Классы Pydantic для запросов и ответов агента.
    *   **`deploy_config.py`**:
        *   **Назначение**: Конфигурация для развертывания агента ежедневной памяти, включая специфичные для среды переменные, настройки ресурсов или правила масштабирования.
        *   **Что в нем**: Параметры развертывания.
    *   **`prompt_config.py`**:
        *   **Назначение**: Содержит конфигурацию промптов, которые используются для взаимодействия с LLM в рамках агента ежедневной памяти.
        *   **Что в нем**: Строки промптов, инструкции для LLM, примеры диалогов.
*   **`daily_memory_agent_tmp20260425_024853/`**:
    *   **Назначение**: Эта директория, вероятно, является временной или тестовой версией агента ежедневной памяти, возможно, созданной для конкретной сессии разработки или отладки. Дата в названии указывает на временный характер.
    *   **Что в нем**: Содержимое, аналогичное `daily_memory_agent/`, но может содержать экспериментальные изменения.
*   **`primary_agent/`**:
    *   **Назначение**: Основной агент, который является центральным компонентом в обработке запросов и генерации ответов. Он оркестрирует взаимодействие с другими сервисами и LLM для выполнения основных функций ассистента.
    *   **`__init__.py`**: Инициализация пакета.
    *   **`agent.py`**:
        *   **Назначение**: Реализация основного агента. Это может быть сложный конечный автомат или логика, которая определяет, какой "use case" (сценарий использования) следует активировать на основе входных данных.
        *   **Что в нем**: Класс PrimaryAgent, методы обработки, логика принятия решений.
    *   **`contracts.py`**:
        *   **Назначение**: Определяет контракты (интерфейсы и модели данных) для взаимодействия с основным агентом.
        *   **Что в нем**: Классы Pydantic для входных и выходных данных агента.
    *   **`deploy_config.py`**:
        *   **Назначение**: Конфигурация развертывания для основного агента.
        *   **Что в нем**: Параметры развертывания.
    *   **`prompt_config.py`**:
        *   **Назначение**: Конфигурация промптов для взаимодействия с LLM в контексте основного агента.
        *   **Что в нем**: Строки промптов, инструкции для LLM, примеры.
*   **`rolling_memory_agent/`**:
    *   **Назначение**: Агент, управляющий "скользящей" или долгосрочной памятью ассистента, которая охватывает более длительные периоды, чем ежедневная память. Это может быть механизм для сохранения контекста между сессиями или для обучения LLM на долгосрочных данных.
    *   **`__init__.py`**: Инициализация пакета.
    *   **`agent.py`**:
        *   **Назначение**: Основная логика агента скользящей памяти. Определяет, как агент обновляет и использует долгосрочный контекст.
        *   **Что в нем**: Класс агента, методы обработки, вызовы LLM и логика сохранения.
    *   **`contracts.py`**:
        *   **Назначение**: Определяет контракты для агента скользящей памяти.
        *   **Что в нем**: Классы Pydantic для запросов и ответов агента.
    *   **`deploy_config.py`**:
        *   **Назначение**: Конфигурация развертывания для агента скользящей памяти.
        *   **Что в нем**: Параметры развертывания.
    *   **`prompt_config.py`**:
        *   **Назначение**: Конфигурация промптов для LLM в контексте агента скользящей памяти.
        *   **Что в нем**: Строки промптов, инструкции для LLM.
