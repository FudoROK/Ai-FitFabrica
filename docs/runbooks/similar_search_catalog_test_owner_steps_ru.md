# План действий владельца: тест B2B-каталога и поиска похожей одежды

Дата: 2026-06-30

Цель: подготовить нормальный тестовый каталог товаров, чтобы проверить, как работает поиск похожей одежды через сайт.

## Простыми словами

Сейчас у нас поиск уже технически работает, но в базе мало реальных товаров. Поэтому результат зависит не от AI, а от того, есть ли в нашей тестовой базе похожие товары.

Твоя задача: загрузить в B2B-каталог несколько реальных тестовых товаров, чтобы система могла искать среди них.

Моя задача после этого: запустить переиндексацию поиска, проверить backend, потом вместе проверить результат на сайте.

## Что нужно подготовить

Минимум для нормального теста: 20 товаров.

Лучше для хорошего теста: 40-60 товаров.

Не надо сразу грузить тысячи товаров. Сначала проверяем качество на маленьком, но разнообразном наборе.

## Какие товары нужны

Нужны товары одежды, желательно из Казахстана, чтобы проверить поиск по городу и стране.

### Обязательные группы

1. Рубашки

- 5-10 товаров.
- Белые, голубые, черные, oversize, классические.
- Это нужно, потому что первый тест уже был с рубашкой.

2. Футболки / лонгсливы

- 5-10 товаров.
- Разные цвета, базовые и с принтом.

3. Брюки / джинсы

- 5-10 товаров.
- Джинсы, классические брюки, широкие брюки.

4. Платья / юбки

- 5-10 товаров.
- Разные фасоны и цвета.

5. Верхняя одежда

- 3-5 товаров.
- Куртка, жакет, пальто, пиджак.

## Какие фото нужны

Для каждого товара минимум 1 фото.

Лучше 2-3 фото на товар, но для первого теста достаточно 1 хорошего фото.

Фото должно быть:

- JPG, PNG или WebP;
- до 10 MB;
- товар хорошо виден;
- без сильного размытия;
- без водяных знаков, если возможно;
- желательно на белом/простом фоне;
- можно фото на модели или предметное фото, но товар должен быть главным объектом.

Не использовать для теста:

- очень тёмные фото;
- фото, где одежда почти не видна;
- коллажи из 5-10 товаров;
- скриншоты с мелким товаром;
- фото с большим количеством посторонних предметов.

## Какие данные нужны по каждому товару

Для каждого товара нужно заполнить:

- название;
- категория;
- описание;
- цена;
- валюта;
- страна;
- город;
- ссылка на товар, если есть;
- доступность;
- регионы доставки.

### Рекомендуемые значения для теста

Страна: `KZ`

Города:

- `Almaty`;
- `Astana`;
- `Shymkent`;
- можно добавить другой город, если товар реально оттуда.

Валюта: `KZT`

Доступность: `in_stock`

Регионы доставки:

- `Almaty`;
- `Astana`;
- `Kazakhstan`.

## Как называть товары

Название должно быть понятное, не просто "товар 1".

Примеры хороших названий:

- `Белая оверсайз рубашка`
- `Голубая классическая рубашка`
- `Черная футболка с круглым вырезом`
- `Синие прямые джинсы`
- `Бежевый жакет`

Плохо:

- `123`
- `test`
- `рубашка`
- `фото`

## Как распределить товары для первого теста

Минимальный набор на завтра:

| Группа | Количество | Пример |
|---|---:|---|
| Рубашки | 8 | белая, голубая, черная, oversize |
| Футболки | 6 | белая, черная, с принтом |

Итого: около 30 товаров.

Это хороший стартовый набор.

## Какой товар выбрать для первого поискового теста

Сначала лучше тестировать рубашку.

Почему:

- у нас уже был живой тест с рубашкой;
- легче понять, правильно ли система ищет похожие товары;
- проще увидеть, учитывает ли система город, цвет, тип одежды и цену.

Первое тестовое фото для поиска:

- белая или светлая рубашка;
- товар хорошо виден;
- без сложного фона;
- лучше не коллаж.

## Пошагово что тебе сделать

### Шаг 1. Подготовить товары

Подготовь 20-30 товаров по группам выше.

Для каждого товара подготовь:

- фото;
- название;
- категорию;
- цену;
- город;
- страну;
- описание.

### Шаг 2. Загрузить товары в B2B-кабинет

Открыть сайт:

`https://fit.aisoulfabrica.com/workspace/business-catalog`

Дальше:

1. Создать или проверить профиль продавца.
2. Добавить товары вручную или через CSV import, если удобнее.
3. Для каждого товара загрузить фото.
4. Отправить товары на проверку.

### Шаг 3. Одобрить товары в админке

Открыть админскую страницу:

`https://fit.aisoulfabrica.com/admin/business-catalog`

Дальше:

1. Проверить товары в очереди.
2. Одобрить нормальные товары.
3. Отклонить мусорные/неполные товары.

Важно: в поиск попадают только товары со статусом `active` + `approved`.

### Шаг 4. Сообщить мне

Когда товары загружены и одобрены, напиши:

`готово, товары загружены и одобрены`

После этого уже нужно будет включить VM.

### Шаг 5. Включить VM

VM нужна только после загрузки и одобрения товаров.

До этого VM можно держать выключенной.

После включения VM я:

- проверю backend;
- запущу переиндексацию каталога;
- проверю, что Qdrant products index заполнен;
- проведу backend smoke;
- потом мы проверим поиск через сайт.

### Шаг 6. Проверить поиск на сайте

Открыть:

`https://fit.aisoulfabrica.com/workspace/similar-search`

Дальше:

1. Загрузить фото одежды.
2. Указать страну `KZ`.
3. Указать город, например `Almaty`.
4. Нажать поиск.
5. Проверить результаты.

## Что мы будем считать успешным тестом

Тест считается успешным, если:

- сайт принимает фото одежды;
- backend запускает Garment Identity;
- поиск возвращает товары из нашего B2B-каталога;
- первыми идут товары из города пользователя;
- похожие товары выглядят логично;
- если точного совпадения нет, система всё равно показывает близкие варианты;
- нет ошибки `500`;
- нет пустого результата при наличии подходящих товаров в каталоге.

## Что может быть неидеально на первом тесте

Это нормально, если:

- результаты похожи не идеально;
- слишком мало вариантов;
- система путает близкие категории;
- по редкой одежде результатов мало.

Почему: качество поиска зависит от количества и качества товаров в базе.

## Что не надо делать завтра

Не надо:

- грузить 500+ товаров;
- подключать реальные магазины клиентов;
- подключать Wildberries/Kaspi/Instagram;
- делать платные внешние API;
- тестировать всё сразу.

Сначала проверяем наш локальный B2B-каталог и поиск по нему.

## Команда, которую буду запускать я после загрузки товаров

```powershell
.venv\Scripts\python.exe scripts/reindex_business_catalog_search.py --limit 1000
```

Эта команда:

- берёт одобренные товары из backend-БД;
- создаёт поисковые embeddings;
- записывает товары в Qdrant;
- готовит каталог для поиска похожей одежды.
