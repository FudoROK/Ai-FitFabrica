# AI FitFabrica Website Implementation Plan

## 1. Objective

Build a production-ready website and workspace frontend for `Ai Fitfabrica` based on the Stitch prototype in `stitch_ai_fitfabrica_prototype`, while following the repository enterprise rules:

- `Next.js + React + TypeScript + Tailwind CSS`
- structured routing and feature-based UI modules
- internal navigation between pages must work
- backend integration points must exist through typed client contracts
- no deploy actions in this stage
- image assets generated by Stitch must be replaced with locally generated assets stored in a separate folder

## 2. Source Design Inventory

- Public marketing:

## 3. Target Frontend Structure

Planned application root:

- `apps/web`

Planned structure:

- `src/app`

  - shared layout
- `src/features`
  - landing
  - workspace

  - pricing
  - contacts
- `src/components`

## 4. Delivery Steps

### Step 1. Planning and route map

- extract route inventory from Stitch
- define route naming and information architecture
- write implementation plan to `docs`

### Step 2. Frontend foundation

- scaffold Next.js application
- configure TypeScript, Tailwind, ESLint, path aliases
- install typography and theme tokens
- define app shell and reusable primitives

### Step 3. Public site implementation

- implement public header, footer and CTA system
- build public pages from the Stitch design system
- replace placeholder `#` links with working route navigation

### Step 4. Workspace implementation

- implement workspace shell with navigation
- build workflow pages and reusable panels
- provide realistic loading, empty, success, warning and error states

### Step 5. Backend integration layer

- create typed API client interfaces
- add service modules and DTOs for future FastAPI integration
- keep integration mocked only at adapter level, not inside UI business logic

### Step 6. Image generation and asset replacement

- identify Stitch-generated visuals
- generate equivalent local assets
- store them in a dedicated frontend asset folder
- replace external URLs with local files

### Step 7. Verification and reporting

- run lint
- run type checks
- run project review pass
- write implementation reports into `docs`

## 5. Constraints

- no static one-file HTML delivery
- no `href="#"`
- no fake backend logic in UI
- no direct AI provider calls from frontend
- no untyped responses in UI code
- no deployment in this stage

## 6. Output of This Stage

Expected result:

- structured website inside the repository
- working internal navigation between public and workspace pages
- locally generated image assets
- implementation reports stored in `docs`
